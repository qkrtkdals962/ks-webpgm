운영체제(OS)


운영체제(Operating System)는 사용자와 컴퓨터 하드웨어 사이에서 중계역할을 하면서, 프로그램의 실행을 관리하고 제어하는 시스템 소프트웨어이다. 이때 운영체제는 모든 컴퓨터 자원을 관리자로서 관리하며 독점한다. 그리고 운영체제는 소프트웨어로서 커널(kernel)이라고 불리는 핵심 코드, 사용자의 컴퓨터 사용을 돕는 UI(User Interface)를 비롯한 도구 프로그램들, 장치를 직접 제어하는 디바이스 드라이버라는 소프트웨어로 구성된다.

역사

컴퓨터가 처음 생길때 부터 운영체제가 있었던 것은 아니다. 1940년대 고정프로그램 컴퓨터가 있었고 1940년대 중반에 기계식을 벗어나 전자식 디지털 컴퓨터가 만들어졌다. 1945년 헝가리 출신의 수학, 물리학, 컴퓨터를 공부한 학자 "폰 노이만"(Von Neumann)이 최초로 하드웨어와 소프트웨어를 분리한 내장형 프로그램 컴퓨터(stored program computer) 개념을 발표했다. 내장형 컴퓨터가 등장한 초기에는 펀치 카드(punched card)에 구명을 뚫어 프로그램을 작성하고, 카드 입력기를 이용하여 프로그램을 메모리에 적재하였고, 출력장치로는 프린터가 있었다.

배치 프로그래밍

과거 1964년전에는 컴퓨터가 작동하는 시간에 비해 셋업에 걸리는 시간이 커서 컴퓨터가 노는시간(idle time)이 많았다. 컴퓨터 활용률을 향상시키기 위해 배치 운영체제(batch operating system)가 개발 되었다. 배치 운영체제의 도입으로 개발자와 관리자가 구분되게 되었다.

배치 처리방식을 살펴보면 개발자가 작성한 프로그램을 펀치카드에 입력하고 입력데크에 올려두면 관리자가 실행을 하게 된다. 여기서 개발자가 작성한 펀치 묶음이 배치(batch)이며 배치 작업(batch job)이라고 불린다. 관리자가 컴퓨터를 작동시키면, 배치 운영체제가 실행되어 자동으로 테이프 장치에 들어있는 배치 작업을 메모리로 읽어 들이고 실행시킨다. 프로그램의 실행 결과는 프린터에 출력되어 관리자가 출력된 결과물을 뜯어 펀치카드와 묶고 출력물을 쌓아두는 곳에 두면 개발자가 가져간다. 배치 시스템은 한번에 한 개의 작업만 실행시켰으므로 CPU가 파일을 읽거나, 쓰거나, 프린터로 출력하는 작업 즉 I/O작업이 발생되면 I/O작업이 끝날때 까지 놀면서 대기할 수밖에 없었다.

다중 프로그래밍

배치 운영체제의 단점으로 인식된 저조한 CPU 활용률을 극대화 하기위해 다중프로그래밍 기법이 제안되었다.

다중 프로그래밍 기법은 여러 프로그램을 메모리에 올려놓고 동시에 실행시키는 기법으로 한 프로그램을 실행하다 I/O가 발생하면 입출력이 완료될 때까지 CPU가 메모리에 적재된 다른 프로그램을 실행하게 하여 CPU의 노는 시간을 줄인다. 이때 다중프로그래밍 기법 아래에서는 메모리에 적재된 프로그램들이 조금씩이지만 모두 동시에 실행되는 셈이다. 하지만 다중프로그래밍에도 문제들이 있었다. 큰 메모리 이슈, 프로그램의 메모리 할당 및 관리 이슈, 메모리 보호 이슈, CPU 스케줄링과 컨텍스트 스위칭 이슈... 등등 수 많은 문제들이 발견되었고 이러한 문제들을 해결하는 과정에서 컴퓨터 시스템에 많은 발전이 이루어 졌다.

시분할 프로그래밍

시분할 프로그래밍은 다중 프로그래밍 기법과는 달리 적재된 여러 프로그램을 시간 단위로 나누어 번갈아 실행시키는 다중 프로그래밍 기법이다.

시분할 운영체제는 배치 처리와 다중프로그래밍 처리에 있는 비대화식 처리방식과, 느린 응답 시간과 오랜 대기 시간 등 2가지의 문제점을 해결하기 위해 개발되었는데 비대화식 처리방식이란 여러 프로그램을 메모리에 적재하고 동시에 실행시키기는 하지만 여러 프로그램들이 특정시간에 모두 처리되며 프로그램 실행 결과도 한참후에 받게 되는 방식이다. 느린 응답 시간과 오랜 대기 시간은 프로그램들이 실행되는 동안 중간 결과에 대해 사용자의 즉각적인 대응이 불가능했다. 그리하여 1962년 MIT교수 McCarty는 CTSS(Computer Time Sharing System)를 개발했다. CTSS는 원격으로 컴퓨터에 접속하여 프로그램 결과를 디스플레이 모니터에 결과를 전송받아 자신의 모니터에서 즉각적으로 확인하고 명령을 내리도록 제안하였다. 그 당시 이 시스템을 시분할 시스템이라고 불렀다.

시분할 시스템은 여러 개의 프로그램을 메모리에 적재하고 1초나 100밀리초(ms) 등 시간 할당량을 정하여 시간 할당량만큼 메모리에 적재된 모든 프로그램에게 돌아가면서 CPU를 할당하고 실행시킨다. 사용자는 터미널(terminal)이라고 하는 키보드, 모니터, 모뎀이라고 하는 간단한 입출력장치를 전화선을 이용하여 메인프레임 컴퓨터에 연결한다. 시분할 운영체제는 사용자 수만큼 사용자와 대화하도록 미리 준비된 프로그램을 실행시킨다. 이 프로그램은 해당 터미널로부터 사용자의 명령을 받아 명령을 처리하고 결과를 터미널에 출력 후 다음 명령을 기다린다. 이때 CPU처리 속도는 사용자가 키를 입력하는 속도에 비교할 수 없이 빠르다. 그러므로 각 사용자는 자신이 내린 명령에 대한 응답이 늦게 온다고 생각하지 않는다.

결론적으로 시분할 운영체제는 다중프로그래밍 기법과, 대화식 처리, 시분할 스케줄링을 결합하여 여러 프로그램을 동시에 실행시키는 운영체제이다. Unix, Linux, Window등 오늘날 대부분의 운영체제가 시분할 운영체제의 기능을 가지고 있다.
